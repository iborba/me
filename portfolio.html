Featured Project

Microservices Migration at TOTVS

Monolith to Microservices Migration

Enterprise SaaS Platform

2017 - 2020

The Challenge

The MVP was built on existing databases where direct connections weren't possible‚Äîonly through SOAP APIs. Average response time was 2-3 seconds per query, involving API authentication, communication latency, and processing large XML data.

The Solution

  • Developed a microservices solution using Node.js with individual databases following DDD model
  • Implemented asynchronous communication between microservices
  • Container orchestration with Kubernetes
  • Ensured easy maintenance and cost-effectiveness

The Results

  • 95% reduction in API response times (from 2-3s to under 200ms)
  • Highly resilient, scalable, and secure architecture
  • Extensively tested codebase
  • Team upskilled in Kubernetes and RabbitMQ

Architecture Overview

The previous MVC application with ASP.NET Razor views was transformed into microfrontends in Angular communicating with a gateway, which called microservices to retrieve and return compressed data‚Äîall within the same network.

This architecture was later replicated across new sectors including healthcare systems and dental clinics.

AI-Powered Document Processing

Document Metadata Extraction Service

EY (Ernst and Young)

2022 - Present

A microservice that provides a way to tackle documents and their metadata. It uses AI to extract metadata from PDF documents, comparing datapoint values against rules and recommendations, generating output for clients.

System Architecture

  • Frontend: Angular + React applications
